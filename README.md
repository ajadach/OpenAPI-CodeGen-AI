## What is this project?

`open_api_codegen_prompt.md` is a **super-prompt for AI models** (e.g. GitHub Copilot, Claude, GPT-4) that replaces traditional code generation tools (swagger-codegen, openapi-generator, etc.) in the process of creating client libraries for REST APIs.

Instead of generating code from templates -- AI guides the user step by step through gathering requirements, and then creates a client library tailored to their needs.

---

## Generated library assumptions

Every library generated by this prompt follows these principles:

| Principle | Description |
|-----------|-------------|
| **Resource-based grouping** | Methods are grouped by API resource (e.g. `client.pets.get_all()`, `client.orders.create(...)`) |
| **2xx responses only** | The library expects an HTTP status code in the 2xx range -- it does not handle error cases |
| **Testing interface** | The library is an interface for API testing -- error handling and negative cases belong in tests, not in the library |

---

## How to use the prompt?

1. Open the `open_api_codegen_prompt.md` file in an AI (e.g. paste its contents as a system instruction or conversation context).
2. AI automatically creates the `status.json` file and starts guiding the user through the steps.
3. Answer the AI questions -- at the end a ready-to-use client library will be generated.

> **Sessions can be resumed.** If you stop mid-way, AI will read `status.json` and return to the last unfinished step -- without repeating already answered questions.

---

## State tracking -- `status.json`

At the beginning of the procedure AI creates a `status.json` file with all keys set to `null`. After each user response the corresponding field is updated.

```json
{
  "swagger_url": null,
  "base_endpoint": null,
  "library_folder": null,
  "openapi_json_confirmed": null,
  "check_sum_openapi_json": null,
  "language": null,
  "language_version": null,
  "http_library": null,
  "client_folder": null,
  "method_prefix": null,
  "modules": null,
  "generation": null,
  "pip_package": null,
  "robot_framework_support": null
}
```

### Session resume behaviour

On every run AI reads `status.json` first:

- **All values `null`** -- fresh start. AI shows the library assumptions and begins from Step 1.
- **Any value is non-null** -- resumed session. AI displays a summary of already collected data, skips already answered questions, and immediately asks the first unanswered question.

## Procedure steps

### Step 1 -- API Information

AI asks for:
- **URL to the Swagger documentation** -- `swagger_url`  
  *(address of the Swagger UI page, saved for reference)*
- **Main REST API endpoint** -- `base_endpoint`  
  *(becomes the `BASE_URL` value in `basic.py`; if no scheme is provided, `https://` is added automatically)*

---

### Step 2 -- Project Structure Configuration

AI asks for:
- **Library folder name** -- `library_folder`  
  *(root folder of the entire project; the Python package will be automatically named `{library_folder}Client`)*

AI automatically:
- Derives `client_folder` as `{library_folder}Client` (without asking the user)
- Creates the directory structure:
  ```
  {library_folder}/
  +-- swagger/          <- user places openapi.json here
  +-- trash_AI/         <- AI temporary files
  ```
- Downloads or accepts a manually placed `openapi.json` file
- Calculates and saves the SHA-256 checksum of `openapi.json`

---

### Step 3 -- Technical Library Configuration

AI asks for:
- **Programming language** -- `language`  
  *(e.g. Python; for Python, Pydantic is used for argument validation)*
- **Language version** -- `language_version`  
  *(e.g. `3.12` -> `requires-python = ">=3.12"` in `pyproject.toml`)*
- **HTTP library** -- `http_library`  
  *(e.g. `requests`; imported in every module and added to dependencies)*
- **Method prefix** -- `method_prefix`  
  - without prefix: `client.add_pet()`  
  - with prefix: `client.post_add_pet()`

Based on the answers AI creates `{library_folder}/{client_folder}/basic.py`:

```python
"""Shared configuration for the API client."""

import requests

BASE_URL: str = "https://{base_endpoint}"
SESSION: requests.Session = requests.Session()
```

---

### Step 4 -- Library Generation

1. AI reads `swagger/openapi.json` and counts the modules (tags).
2. The user chooses whether to generate all modules or only selected ones.
3. AI generates modules one by one, tracking the state in `status.json` (`pending` -> `in_progress` -> `done`).

#### Generated project structure

```
{library_folder}/
+-- swagger/
|   +-- openapi.json
+-- trash_AI/
+-- {client_folder}/
    +-- __init__.py
    +-- basic.py              <- BASE_URL and SESSION
    +-- client.py             <- main client: client.pet.get_all()
    +-- AI/
    |   +-- status.json       <- copy of generation artifacts
    |   +-- swagger/
    |       +-- openapi.json
    +-- modules/
        +-- __init__.py
        +-- pet.py            <- Pet class with methods for /pet resource
        +-- store.py
        +-- user.py
```

#### Code generation rules

- Naming convention: tag `pet` -> file `pet.py`, class `Pet`
- Module classes have **no constructor parameters** -- configuration comes from `basic.py`
- Every class and method must have a **docstring** with a description from the `summary` field in `openapi.json`
- Method docstring format: description, `Args:` section, `Returns:` section
- Method order in classes follows the endpoint order in `openapi.json`
- For Python: **Pydantic** (`@validate_call`) for argument validation

---

### Step 5 -- Robot Framework Support *(Python only)*

AI asks whether to add Robot Framework support.

If **yes**:
- `ROBOT_LIBRARY_SCOPE = "SUITE"` and `from robot.api.deco import keyword` are added to every module
- Every public method gets a `@keyword("<Keyword Name>")` decorator  
  *(e.g. method `post_add_pet` -> keyword `"Post Add Pet"`)*
- Modules become standalone RF libraries: `Library    {client_folder}.Pet`

Example Robot Framework usage:
```robot
*** Settings ***
Library    PetStore2Client
Library    PetStore2Client.Pet

*** Test Cases ***
Example
    ${pet}=    Evaluate    {"name": "doggie", "photoUrls": [], "status": "available"}
    PetStore2Client.Pet.Post Add Pet    ${pet}
```

---

### Step 6 -- pip Package *(Python only)*

AI asks whether to prepare the library as a pip package.

If **yes**:
- `pyproject.toml` is created with `[project]`, `[build-system]`, and `[tool.setuptools.packages.find]` sections
- `README.md` is created with a description of the library, methods, and usage example
- The package is built: `python -m build`
- `.whl` and `.tar.gz` files appear in the `dist/` folder

If **no**:
- AI explains how to use the library locally without pip:
  ```python
  import sys
  sys.path.insert(0, "/path/to/{library_folder}")
  from {client_folder}.client import <ClientClass>
  client = <ClientClass>()
  ```

---

### Step 7 -- Archive Artifacts and Summary

AI copies generation artifacts into the client package folder:
- `{library_folder}/swagger/openapi.json` -> `{library_folder}/{client_folder}/AI/swagger/openapi.json`
- `status.json` -> `{library_folder}/{client_folder}/AI/status.json`

A final success summary is then displayed with details of all generated elements.

---

## Built-in AI rules

| Rule | Description |
|------|-------------|
| No unnecessary `print` | AI does not add print statements to generated code |
| Language standards | Code is written according to the standards of the chosen language (e.g. PEP8 for Python) |
| Docstrings everywhere | Every class and method has a docstring with description, parameters, and return value |
| Pydantic (Python) | Argument validation via `@validate_call` |
| Never re-ask | Never asks about values already saved in `status.json` |
| Dynamic work | After every break AI returns to the last unfinished step |

### PetStore2

A complete Python client library generated for the [Swagger Petstore v2](https://petstore.swagger.io/) API.

| Property | Value |
|----------|-------|
| Base endpoint | `https://petstore.swagger.io/v2` |
| Language | Python >= 3.12 |
| HTTP library | `requests` |
| Modules | `pet`, `store`, `user` |
| Robot Framework support | yes |
| pip package | yes |

#### Installation

> **Note:** The `dist/` folder is not included in the repository. You need to build the package yourself before installing it.

Navigate to the `Example_Results/PetStore2/` directory and run:

```bash
pip install build
python -m build
pip install dist/PetStore2Client-1.0.0-py3-none-any.whl
```

#### Quick start

```python
from PetStore2Client.client import PetStore2Client

client = PetStore2Client()
pet = client.pet.get_pet_by_id(1)
print(pet)
```
